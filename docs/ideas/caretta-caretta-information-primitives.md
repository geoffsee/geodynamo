# Caretta Caretta Information Primitives

This note experiments with translating the vector model in
[Caretta Caretta Field Vectors](./caretta-caretta-field-vectors.md) into
information-theoretic terms. It is an abstraction for thinking about sensory
processing, not a claim that the turtle's receptor mechanism is known at this
level.

## Signal Model

The geomagnetic field can be treated as an environmental signal generated by a
hidden world state.

```text
S = {latitude, longitude, time, local anomalies}
B = g(S)
Y = B + N
Z = phi(Y) = [F, H, I, A]
U = policy(Z, M, C)
```

| Symbol | Information primitive | Interpretation |
| --- | --- | --- |
| `S` | Source state | The hidden ecological/geographic state that produces a local magnetic field. |
| `B` | Signal vector | The structured physical message available in the environment. |
| `Bx`, `By`, `Bz` | Signal coordinates | Channel-specific components of the physical signal. |
| `N` | Noise | Sensor noise, geomagnetic disturbance, body orientation error, or local anomaly. |
| `Y` | Observation | The received field after biological and environmental noise. |
| `phi()` | Encoder / feature map | The transformation from raw observation into compact sensory variables. |
| `Z` | Code / feature vector | The turtle-usable representation derived from `Y`. |
| `M` | Memory / codebook | Inherited route templates or learned magnetic signatures. |
| `C` | Context | Current behavior, developmental state, ocean current, light, odor, or wave cues. |
| `U` | Action message | The selected swimming vector or orientation response. |

## Symbol Mapping

The original mathematical symbols can be reinterpreted as compression and
decision variables.

| Symbol | Information role | Notes |
| --- | --- | --- |
| `B` | Raw environmental message | High-dimensional enough to carry both map and compass information. |
| `F` | Compressed intensity statistic | A scalar feature that may help identify geographic region. |
| `H` | Intermediate projection statistic | Useful for deriving inclination and separating vertical from horizontal structure. |
| `I` | Positional feature | Inclination can function as a latitude-like cue in the magnetic map. |
| `A` | Directional feature | Azimuth or bearing can support compass-like orientation. |
| `[F, I]` | Map codeword | A compact magnetic signature for place recognition. |
| `[A, I]` | Compass codeword | A compact directional signature for heading control. |
| `target_signature` | Stored codeword | A learned or inherited reference pattern. |
| `error` | Distortion / prediction error | Difference between current sensory code and a stored or expected code. |
| `policy()` | Decoder / controller | Converts encoded sensory information into motor action. |

The overloaded letter `H` should be kept distinct from Shannon entropy. In this
note, `H` means horizontal magnetic magnitude. Use `H_shannon(X)` or
`entropy(X)` for uncertainty.

## Possible Pipeline

One compact way to describe the process is:

```text
world source -> physical field -> receptor channel -> sensory code -> memory match -> action
S            -> B              -> Y                -> Z            -> error        -> U
```

Under this framing, the turtle does not need a complete physical reconstruction
of the magnetic field. It only needs enough mutual information between `Z` and
task-relevant state variables to choose useful behavior.

```text
maximize useful_information = MI(Z; route_state)
minimize irrelevant_variation = noise(Z | route_state)
choose U = argmax expected_navigation_value(U | Z, M, C)
```

Here, `MI()` means mutual information. It is written this way to avoid conflict
with `I`, the inclination angle.

## Experimental Ideas

1. Treat `[F, I]` as a place-recognition code.

   Simulate fields along a route and ask how many distinct route regions can be
   separated by intensity and inclination alone. The information question is:
   how much does the two-scalar code reduce uncertainty over location?

2. Treat `[A, I]` as a heading-control code.

   Rotate the animal body frame relative to the field and measure whether the
   encoded bearing remains stable enough to select a swimming direction. The
   information question is: how much bearing information survives receptor and
   body-orientation noise?

3. Treat `policy(error, context)` as a decoder.

   Compare simple controllers: nearest stored signature, gradient following,
   inherited route table, and context-gated switching between cues. The
   information question is: which decoder extracts navigational value from the
   same compressed signal?

4. Treat magnetic anomalies as channel noise.

   Add perturbations to `B` and observe when `[F, I]` maps to the wrong region
   or `[A, I]` maps to the wrong heading. The information question is: which
   features are robust, and which are fragile?

## Working Hypothesis

The field vector `B` is the physical carrier. The derived features `F`, `H`,
`I`, and `A` are lossy encodings. The biologically useful object is not the
full vector but a task-shaped sensory code `Z`, because navigation only requires
enough information to classify place, maintain route, and choose the next swim
vector.
