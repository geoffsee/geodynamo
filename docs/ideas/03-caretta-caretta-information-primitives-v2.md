# Caretta Caretta Information Primitives (v2)

This note revises [v1](./02-caretta-caretta-information-primitives.md) by addressing
peer-review feedback: naming the framework (Information Bottleneck), defining
the task variable, splitting the noise model, and tightening the
sufficient-statistic claim. It remains an abstraction for thinking about sensory
processing, not a claim that the turtle's receptor mechanism is known at this
level.

## Signal Model

The geomagnetic field is treated as an environmental signal generated by a
hidden world state. Stochasticity is split by biological source rather than
collapsed into a single additive term.

```text
S      = (latitude, longitude, time, local anomalies)   // hidden world state
T      = task variable derived from S                   // e.g. natal-beach latitude band
B      = g(S)                                            // physical field at the animal
Y      = R_body( B + N_field ) + N_receptor              // received signal
Z      = phi(Y)                                          // sensory code (deterministic encoder)
U      = policy(Z, M, C)                                 // action
```

| Symbol | Information primitive | Interpretation |
| --- | --- | --- |
| `S` | Source state | Hidden ecological / geographic state that produces a local magnetic field. Ordered tuple. |
| `T` | Task variable | The thing the turtle needs to predict or preserve information about (e.g. latitude band, distance-to-target, geographic cell index, goal bearing). Must be named for any MI claim to be well-defined. |
| `B` | Signal vector | Structured physical message in the environment. |
| `Bx`, `By`, `Bz` | Signal coordinates | Channel-specific components of the physical signal. |
| `N_field` | Field-level noise | Geomagnetic disturbance, Sq variation, storms, local anomalies. Temporally correlated. |
| `R_body` | Body-frame rotation | Maps world-frame field into the animal's body frame; corrupts bearing but not magnitude or inclination. |
| `N_receptor` | Receptor noise | Transduction noise; affects gain on `F` and `I` differently than on `A`. |
| `Y` | Observation | Received field after all three noise sources. |
| `phi()` | Encoder / feature map | Deterministic map from observation to compact sensory variables. All stochasticity is absorbed upstream in `N_field`, `R_body`, `N_receptor`. |
| `Z` | Code / feature vector | Turtle-usable representation derived from `Y`. |
| `M` | Memory / codebook | Inherited route templates or learned magnetic signatures. Parameterization (discrete codebook vs continuous manifold) must be specified per experiment. |
| `C` | Context | Current behavior, developmental state, ocean current, light, odor, wave cues. |
| `U` | Action | Selected swimming vector or orientation response. |

## Symbol Mapping

The mathematical symbols from the vector model are reinterpreted as candidate
sufficient statistics and decision variables.

| Symbol | Information role | Notes |
| --- | --- | --- |
| `B` | Raw environmental message | High-dimensional enough to carry both map and compass information. |
| `F` | Compressed intensity statistic | Scalar feature that may help identify geographic region. |
| `H` | Intermediate quantity (not in `Z`) | `H = sqrt(Bx^2 + By^2)`. Used only to compute `I`; redundant given `F` and `I` since `H = F·cos(I)`. Excluded from the sensory code to keep `Z` non-redundant. |
| `I` | Positional feature | Inclination angle; latitude-like cue in the magnetic map. |
| `A` | Directional feature, **body frame** | Magnetic bearing computed in the animal's body frame, not world frame. Without a body-frame interpretation, `A` carries no heading information. |
| `[F, I]` | Map codeword | Candidate sufficient statistic of `B` for geographic-region identification. |
| `[A, I]` | Compass codeword | Candidate sufficient statistic of `B` for heading control. |
| `target_signature` | Stored codeword in `M` | Learned or inherited reference pattern. |
| `distortion d(Z, Z*)` | Code-to-target distance | Used by nearest-signature decoders. |
| `prediction_error Z - E[Z | M, C]` | Predictive residual | Used by predictive-coding decoders. Distinct from distortion; do not conflate. |
| `policy()` | Decoder / controller | Maps encoded sensory information to motor action. |

Notation conflicts noted in v1 still hold: `H` here means horizontal magnetic
magnitude (now demoted to intermediate), not Shannon entropy. Use `H_shannon(X)`
or `entropy(X)` for uncertainty. Mutual information is written `MI(X; Y)` to
avoid clashing with inclination `I`.

## Framework: Information Bottleneck

The objective the turtle's sensory pipeline is being modeled against is the
Information Bottleneck (Tishby, Pereira, Bialek):

```text
phi* = argmin_phi   MI(Y; Z)  -  beta * MI(Z; T)
```

That is, find the encoder `phi` that produces a compact code `Z` (low `MI(Y; Z)`)
while preserving information about the task variable `T` (high `MI(Z; T)`). The
parameter `beta` trades compression against task-relevance.

In this framing:

- `[F, I]` and `[A, I]` are **candidate sufficient statistics** of `B` for two
  different task variables (place identification vs heading selection).
- Whether they *are* sufficient is an empirical question: does conditioning on
  `[F, I]` give the same behavior as conditioning on the full `B`?
- The Information Bottleneck curve (achievable `MI(Z; T)` vs `MI(Y; Z)`) gives a
  principled way to compare encoders.

## Pipeline

```text
world source -> physical field -> body-framed receptor -> sensory code -> memory match -> action
S            -> B              -> Y                    -> Z            -> error        -> U
```

The turtle does not need a complete physical reconstruction of the field. It
needs enough mutual information between `Z` and the task variable `T` to choose
useful behavior.

```text
maximize task-relevant info  = MI(Z; T)
minimize irrelevant variation = MI(Z; Y) - MI(Z; T)
choose U                     = argmax expected_navigation_value(U | Z, M, C)
```

## Known limit: the magnetic-doubles problem

`MI([F, I]; location)` is bounded by geometry, not just by receptor quality. In
large regions of ocean, the isolines of total intensity `F` and inclination `I`
run nearly parallel, so distinct geographic points share nearly identical
`[F, I]` signatures ("magnetic doubles"). Any experiment treating `[F, I]` as a
place-recognition code should expect this as the dominant limiting factor, not
as a finding. It also motivates the role of `C` (non-magnetic context cues) in
disambiguation.

## Experimental Ideas

Each experiment must commit to (a) a definition of `T`, (b) a parameterization
of `M`, and (c) a regionalization of space if "wrong region" is the outcome
measure.

1. **Place-recognition code: `[F, I]`.**

   Simulate fields along a route. Define `T` = geographic cell index over a
   chosen tessellation. Parameterize `M` as either a finite codebook of stored
   `[F, I]` signatures or a continuous Gaussian-mixture prior. Measure
   `MI([F, I]; T)` and compare against `MI(B; T)`. The magnetic-doubles geometry
   sets an upper bound; the experiment quantifies how close the two-scalar code
   gets to that bound.

2. **Heading-control code: `[A, I]`.**

   Define `T` = world-frame goal bearing. Rotate the body frame by `R_body` and
   measure whether the encoded bearing remains stable enough to select a
   swimming direction. Information question: how much of `MI([A, I]; T)`
   survives realistic body-orientation noise?

3. **Decoder comparison.**

   Compare nearest-stored-signature, gradient-following, inherited route table,
   and context-gated switching between cues. For each decoder, measure expected
   navigation value at fixed `MI(Z; T)`. Information question: which decoder
   extracts the most navigational value from the same compressed signal?

4. **Anomalies as channel noise.**

   Inject perturbations selectively into `N_field`, `R_body`, and `N_receptor`
   and observe which codewords degrade. Prediction: `[A, I]` is sensitive to
   `R_body`; `[F, I]` is not. `F` is sensitive to receptor gain noise; `I` is
   relatively robust. Information question: which features are robust under
   which channel?

## Working Hypothesis

The field vector `B` is the physical carrier. The candidate codewords `[F, I]`
and `[A, I]` are lossy, low-dimensional statistics of `B`.

The **sufficiency claim** — the empirically testable version of v1's working
hypothesis — is:

> Behavior conditioned on `Z` is indistinguishable from behavior conditioned on
> the full vector `B`, even though `dim(Z) ≪ dim(B)`.

If that holds for `Z = [F, I]` on map tasks and `Z = [A, I]` on compass tasks,
then the two-codeword decomposition is not just a convenient abstraction — it
is the operative sensory representation for navigation.

## Changes from v1

- Defined `T` (task variable) explicitly; all MI expressions now reference it.
- Named the framework: Information Bottleneck.
- Split `N` into `N_field`, `R_body`, `N_receptor`; these have distinct effects
  on `[F, I]` vs `[A, I]`.
- Demoted `H` from a code feature to an intermediate quantity (redundant given
  `F` and `I`).
- Specified that `A` is in the body frame.
- Distinguished distortion `d(Z, Z*)` from prediction error.
- Flagged the magnetic-doubles geometric limit on `MI([F, I]; location)`.
- Restated the working hypothesis as a sufficiency claim rather than a
  definitional one.
- Required each experiment to specify `T`, the parameterization of `M`, and any
  regionalization used for outcome measurement.
